title: SQL Review

<Note>
SQL Review is skipped for statements exceeding 1MB to prevent excessive processing time.
</Note>

SQL Review automatically validates SQL statements against best practices and catches potential issues before they reach production. This eliminates manual review bottlenecks and ensures consistent code quality across your database changes.

## Setting Up SQL Review

Configure SQL Review in two steps:

1. **Define Rules**: Select [review rules](/sql-review/review-rules) for each database engine type
2. **Create Policies**: Build [review policies](/sql-review/review-policy) and assign them to specific environments or projects

![bb-sql-review-policy](/content/docs/sql-review/bb-sql-review-policy.webp)

## Automated Validation

When developers submit database changes through the [change workflow](/change-database/change-workflow/), SQL Review automatically validates the statements and flags any violations before human review:

## How it Works

<Card title="Tutorial: SQL Review with Bytebase GUI" icon="graduation-cap" href="/tutorials/sql-review-gui" horizontal />

SQL Review runs automatically whenever you create database changes. The review process checks for:

- Compliance with configured [SQL Review Rules](/sql-review/review-policy)
- Database connectivity issues
- SQL syntax errors
- Missing migration schema (Bytebase's internal migration tracking)
