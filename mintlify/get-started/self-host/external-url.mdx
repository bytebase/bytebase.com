---
title: Configure External URL
---

The external URL determines how users access your Bytebase instance. This configuration is essential for features like [GitOps](/vcs-integration/overview), [SSO](/administration/sso/overview), and [SCIM](/administration/scim/overview).

## Configuration Methods

### Option 1: Command Line

Pass the `--external-url` flag when starting Bytebase:

```bash
./bytebase --external-url "https://bytebase.example.com"
```

<Note>
A Fully Qualified Domain Name (FQDN) is preferred over an IP address.
</Note>

### Option 2: Web UI

1. Navigate to **Settings** â†’ **General** under **Workspace**
2. Enter your External URL (e.g., `https://bytebase.example.com`)
3. Click **Update**

![external-url](/content/docs/get-started/install/external-url.webp)

## Important Notes

- Include the port number if not using standard ports (80/443)
  - Example: `http://bytebase.example.com:8080`
- The setting persists after initial configuration
- For production, use a reverse proxy (Nginx, Caddy) instead of exposing Bytebase directly

## Additional Configuration

**GitOps Webhook URL**: If your public endpoint differs from the internal one, configure this separately via the UI. It defaults to the External URL if not set.

**Testing**: For local testing, you can use [ngrok](https://ngrok.com) to create a temporary external URL.
